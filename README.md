# StyleConv
StyleConv 是一个文风用词改写工具，支持配置个性化语料库，用于把原始文本改写成更自然、更贴近目标风格的表达。
## 概览
StyleConv 面向需要“改写”而不是“重写一切”的场景。它支持基础去模板化改写，也支持基于个人语料生成风格画像与词汇映射，在保留原意的前提下，让结果更接近指定写作习惯。项目采用本地数据工作流：persona、语料和分析结果保存在本地目录，模型配置保存在浏览器本地，适合个人写作实验、风格模仿和私有语料驱动的表达优化。
## 快速开始
clone
```bash
npm install
npm run dev
```
访问: `http://localhost:3000`

## 使用教程

### 1. 配置模型

启动项目后，先进入 `API 设置` 页面，填写以下信息：

- `Base URL`：兼容 OpenAI 接口的服务地址
- `API Key`：你的模型服务密钥
- `Model`：要使用的模型名称

填写完成后点击“保存配置”。这些配置只会保存在当前浏览器本地，不会自动写入仓库。

### 2. 创建 Persona

进入 `语料角色` 页面，点击“新建语料角色”，填写：

- 标题：这个 persona 的名称
- 简介：这个 persona 的风格描述

创建后会生成一个新的语料角色卡，用来承载该风格对应的语料、画像和映射规则。

### 3. 上传语料

在 persona 卡片中点击“添加语料”，上传 `txt` 或 `md` 文件。系统会把这些文本作为该 persona 的风格依据。

建议语料尽量满足以下条件：

- 来自同一种写作风格
- 文本内容尽量完整，不要过短
- 尽量减少杂乱格式、广告语或无关内容

### 4. 重建 Persona

上传语料后，可以点击“重绘”重新分析 persona。这个过程会结合当前语料生成：

- 风格总结
- 常用表达习惯
- 词汇映射关系

如果你更新了语料，建议重新执行一次重建。

### 5. 查看和编辑 Persona 详情

点击 persona 卡片可以进入详情页，查看当前 persona 的：

- 风格总结
- 统计信息
- 词汇映射表

你可以直接修改画像总结和映射内容，然后点击“保存修改”。

### 6. 执行改写

进入 `工作台` 页面后：

1. 在左侧输入原始文本
2. 填写“附加约束”，例如希望更口语化、更简洁或更克制
3. 选择 Persona
4. 选择改写模式
5. 点击“生成”

生成完成后，右侧会显示改写结果，你也可以直接点击“复制”。

### 7. 改写模式说明

- `基础改写`：不依赖 persona，适合通用去模板化表达
- `贴合 persona`：按照选中 persona 的整体风格进行改写
- `映射增强`：更强调 persona 中沉淀出来的词汇映射和表达替换习惯

### 8. 数据存放位置

- `data/personas/`：本地 persona、语料和分析结果
